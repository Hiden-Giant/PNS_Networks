# Phase 0 요구사항 기준선

## 목적

`platform-implementation-plan.md`를 실제 개발 백로그로 전환하기 위한 최초 기준선이다. 제안서의 잘못된 WMS 항목은 제외하고 포워딩 영업, 운임, 스케줄, 견적 업무만 다룬다.

## 확정 사항

- 입고 확정, 로케이션, 재고 수량, 창고·매장 역할은 구현하지 않는다.
- 라우트, 스케줄, 운임은 독립 마스터로 관리한다.
- 매입 원가, 부대비용, 마진·할인, 판매가를 분리한다.
- 견적 확정 시 계산 근거를 스냅샷으로 저장한다.
- UI 권한과 별개로 서버에서 모든 권한을 다시 검증한다.
- Next.js 16 App Router와 Server Component를 기본으로 사용한다.
- 관계형 DB는 PostgreSQL을 우선안으로 한다.
- 외부 API, 엑셀 반영, 이메일은 비동기 작업으로 분리한다.

## 1차 구현 에픽

### E01 사용자와 접근 제어

- 포워딩 역할 정의
- 로그인 세션
- 역할·권한 서버 검증
- 사용자 활성·비활성
- 접근 거부 및 감사 기록

### E02 코드북

- 지역, 국가, 항구, 공항
- 선사, 항공사
- 운송 방식, 통화, 단위
- Incoterms
- 부가 서비스와 비용 코드

### E03 라우트

- 라우트 등록·조회·수정·비활성화
- 출발지와 도착지 조합 중복 검증
- 해상·항공·육상 구분

### E04 스케줄

- 라우트와 선사 연결
- ETD, ETA, 판매 기간
- 초안, 오픈 예정, 판매 중, 보류, 만료, 취소 상태

### E05 운임

- 매입 운임과 부대비용
- 과금 단위와 통화
- 유효기간과 데이터 출처
- 마진 규칙과 고객 특가
- 엑셀 업로드와 행 단위 검증

### E06 조회와 견적

- 유효 스케줄·판매가 검색
- 고객과 화물 조건 입력
- 견적 라인 계산
- 승인, 발송, 버전과 상태 이력
- PDF 견적서

### E07 연동과 운영

- 연동 업체와 실행 설정
- 원본 데이터 보존과 표준 모델 변환
- 실패 재처리와 모니터링
- 접속·데이터 변경 감사 로그

## 구현 순서

1. 도메인 타입과 권한 모델
2. DB 도구 선정 및 스키마 마이그레이션
3. 인증과 서버 권한 경계
4. 코드북 CRUD
5. 라우트·스케줄·운임 CRUD
6. 현재 목업 검색을 DB 검색으로 교체
7. 고객·견적 업무 흐름
8. 외부 연동과 운영 안정화

## 구현 전 확인이 필요한 업무 결정

- 기존 사내 인증 연동 여부와 계정 정책
- 기준 통화와 환율 적용 기준
- 운송 단위별 반올림 규칙
- 마진 규칙 우선순위와 할인 승인 한도
- 견적번호 채번 규칙과 유효기간 기본값
- 기존 운임 엑셀의 실제 열 구조
- 첫 외부 연동 대상과 API 사양
- 감사 로그 및 영업 데이터 보존 기간

미확정 항목은 데이터 구조가 확장 가능하도록 설계하되, 임의의 업무 규칙을 운영 로직으로 확정하지 않는다.
